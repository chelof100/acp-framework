=== Findings ===
CW_approved = N × k₀  (linear): each agent independently contributes
k₀=1 approved op before anomaly escalates (Rule3) then blocks (Rule1).

TTB_reqs    = N×(k₀-1)+1 (round-robin): more agents delay the first
block in total-request count, but do not reduce per-agent exposure.

Design boundary: PatternKey = SHA-256(agentID | cap | res) ensures
per-agent scope. N agents targeting the same resource accumulate risk
independently. Total approved operations = N × k₀ (linear in N).

ACP bounds coordination per-agent. Resource-level attribution against
coordinated access requires an additional mechanism beyond RISK-3.0
per-agent admission control (identified as future work).
