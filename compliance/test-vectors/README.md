# ACP Test Vectors

This directory contains official test vectors for ACP conformance testing, following the format defined in [`../compliance/ACP-TS-1.1.md`](../compliance/ACP-TS-1.1.md).

---

## Format

Each vector is a JSON file with four sections:

```json
{
  "meta":     { "id", "acp_version", "layer", "conformance_level", "description", "severity" },
  "input":    { ... object to evaluate ... },
  "context":  { "current_time", "revocation_list", "trusted_issuers", ... },
  "expected": { "decision", "error_code" }
}
```

All vectors use `context.current_time` — **never system time**.

Fixed reference timestamp used across all vectors: `1700000000` (2023-11-14T22:13:20Z).

---

## Test Keys

All positive vectors use real Ed25519 signatures generated by `impl/go/cmd/acp-sign-vectors`.
Signatures are not placeholders — they are cryptographically valid and verifiable.

### Institution key (CORE/DCMA capability signatures)

```
# TEST KEY A — DO NOT USE IN PRODUCTION
Seed:    9d61b19deffd59985ba34a442fa1c54cd044c9c565b66f2699171d66c9682252
DID:     did:key:z6MkekQTaq7vjX7Vdy6pxabbjgkauuzprRGbBWNAXDs1NZdQ
```

This key signs `capability.signature` fields in CORE and DCMA vectors.

### Agent keys (DCMA delegation and HP PoP signatures)

```
# did:example:agent-alice  (RFC 8037 key B)
Seed:    4ccd089b28ff96da9db6c346ec114e0f5b8a319f35aba624da8cf6ed4d0bd6d8
Public:  YbpNfZpGun1v9UCEqxfhWkexj4WADnmk_YsPkfFbry4  (base64url)

# did:example:agent-bob
Seed:    c5aa8df43f9f837bedb7442f31dcb7b166d38535076f094b85ce3a2e0b4458f7

# did:example:agent-a3
Seed:    f5e287302bfa55e4ae4b18c2f793a3a09e74e5c9547e80d22b55d3a51b3ddbc1
```

These keys sign `delegation.delegation_signature` (DCMA) and `pop_token.sig` (HP) fields.

Negative test vectors with wrong signatures use `aGVsbG8gd29ybGQ` (base64url of "hello world") as a deliberately invalid value.

## Token Schema: ACP-TS-1.1 vs ACP-CT-1.0

Test vectors use the **ACP-TS-1.1 test schema**, which differs from the production **ACP-CT-1.0 schema**.

| Field | ACP-TS-1.1 (test vectors) | ACP-CT-1.0 (production tokens) |
|---|---|---|
| Issuer | `issuer` | `iss` |
| Capability set | `action_set` (string) | `cap` (array of strings) |
| Token ID | `jti` | `nonce` |
| Signature | `signature` | `sig` (under `proof`) |

The IUT evaluator (`cmd/acp-evaluate`) operates on ACP-TS-1.1 test vectors.
The production token verifier (`pkg/tokens`) operates on ACP-CT-1.0 tokens.
These two pipelines are intentionally separate — do not mix schemas.
